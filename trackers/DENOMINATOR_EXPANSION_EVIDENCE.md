# Denominator expansion evidence

This tranche makes the conversion percentage materially more honest by admitting the remaining coarse repo-scope Haskell inventory rows into the governed active matrix.

## What changed

- The active capability matrix now includes every row from `trackers/HASKELL_CAPABILITY_SEED.csv`.
- Already-implemented and already-verified fine-grained rows remain unchanged.
- Unimplemented repo-scope families remain explicitly visible as `mapped_only` rows instead of living outside the denominator.

## Why this matters

The previous 100% figure only reflected the already-admitted bootstrap matrix. That was structurally valid but too flattering for planning.

This tranche shifts the project onto a more useful control basis: if a major reader, writer, runtime, Lua, server, or citeproc family exists in the Haskell inventory but is not implemented in Python, it now visibly depresses the percentage instead of hiding outside it.
